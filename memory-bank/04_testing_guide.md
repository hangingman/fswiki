# テスト戦略の文書化

## 概要

現在のところ、FSWikiプロジェクトにおけるテスト戦略は明確に文書化されていません。また、一般的なPerlのテストファイル命名規則（`*.t`）に従うテストファイルは、プロジェクト内で見つかりませんでした。

## テストの実行方法

### ユニットテスト

*   **フレームワーク:** `Test::More` を使用しています。これはPerlの標準的なテストモジュールであり、将来的なMojoliciousへの移行も考慮されています。
*   **テストファイルの配置:** テストファイルはプロジェクトルートの `t/` ディレクトリ以下に配置します。ファイル名は `NN_description.t` の形式（例: `00_compile.t`）とします。
*   **実行方法:** `Makefile` の `test` ターゲットを使用します。
    ```bash
    make test
    ```
    このコマンドは、開発用Dockerコンテナ内で `prove -l -Iplugin t` を実行し、`t/` ディレクトリ以下のすべてのテストを実行します。テスト実行前に依存関係が自動的にインストールされます。

### スモークテスト (`t/00_compile.t`)

`t/00_compile.t` は、アプリケーションの主要なモジュールがコンパイル可能であることを確認するスモークテストです。これは、Perlのモジュール間の依存関係を考慮し、`WikiApplication.pm` が正常に `use` できることを確認することで、アプリケーション全体の基本的な健全性をチェックします。

## 今後の課題

*   ユニットテスト、結合テスト、E2Eテストの導入を検討し、コードの品質と信頼性を向上させる。
*   テストの実行方法、テストデータの管理、テストカバレッジの測定方法などを文書化する。
